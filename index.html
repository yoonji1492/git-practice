git flow

main : 내가 실제 사용자에게 서비스 할 것들 release : 실제 배포되는 브랜치 develop : 내가 현재 개발 중인 브랜치 -> git branch "브랜치 명" -> git checkout -b "브랜치 명" feature : 개발 중인 것들에서도 기능을 개발하는 브랜치 hotfix : release 중 오류가 나면 그걸 빠르게 버그 수정을 해서 main에 합쳐 배포하는 브랜치 git merge : 합치는 명령어 git pull : 원격 저장소에서 내용을 가져오는 명령어 git을 잘 사용하면 재미있습니다.
